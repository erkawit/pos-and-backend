# 🚀 คู่มือการนำคู่หู POS ขึ้นระบบจริงบน Firebase และ Vercel (Deployment Guide)

คู่มือนี้จัดทำขึ้นโดยเฉพาะสำหรับคุณ เพื่อเตรียมความพร้อมในการนำโปรเจกต์ระบบดุลบริหารหน้าร้าน คลังพัสดุ และฐานสมาชิกอัจฉริยะ (Smart POS & CRM) ไปรันบนแพลตฟอร์มของคุณเองแบบถาวร (Firebase + Vercel)

---

## 📁 1. แฟ้มข้อมูลที่เราเตรียมไว้ให้พร้อมแล้วในระบบ
ในโฟลเดอร์รากโปรเจกต์ของคุณ เราได้เตรียมโครงสร้างไฟล์สำคัญสำหรับรันบนโปรดักชันไว้ทั้งหมดแล้ว:
*   `vercel.json` (ตั้งค่าระบบ Routing สำหรับ Vercel ให้หน้าร้านทำงานแบบ Single-Page Application ป้องกันปัญหาหน้ารีเฟรชแล้วเกิดอาการ 404 บล็อกการเปลี่ยนหน้าเส้นทาง)
*   `firestore.rules` (กฎความปลอดภัยสไตล์ Zero-Trust ช่วยปกป้อง CRM สมาชิก บันทึกบิลขาย และข้อมูลสินค้าสินค้าหลักไม่ให้โดนแฮกหรือปลอมแปลง)
*   `firebase-blueprint.json` (พจนานุกรมโครงสร้างข้อมูลของ Firestore ช่วยให้คุณเห็นสคีมาข้อมูลของแต่ละคอลเลกชันได้ง่าย)

## 🛠️ ขั้นตอนที่ 1: การตั้งค่าฝั่ง Firebase และแก้ไขข้อผิดพลาดระบบโดเนมล็อกอิน

### ปัญหาที่พบ ณ ตอนนี้: `auth/unauthorized-domain`
เหตุผลที่คุณพบข้อผิดพลาดนี้ เกิดจาก **Firebase Authentication จำกัดสิทธิ์หน้าต่าง Google Sign-in** ให้เฉพาะโดแมนที่ได้รับการรับรองความปลอดภัยล่วงหน้าเท่านั้น หากเรานำโค้ดไปรันบน URL พรีวิวของ AI Studio หรือโดเมน Vercel ของคุณ (e.g. `your-app.vercel.app`) ระบบ Google จะบล็อกไว้ทันที

### วิธีแก้ปัญหาระบบและตั้งค่า Firebase:
1.  เปิดหน้า **[Firebase Console](https://console.firebase.google.com/)** ของคุณ
2.  กดสร้างโปรเจกต์ใหม่ (หรือเลือกใช้โปรเจกต์เดิมที่คุณดูแล)
3.  เปิดใช้งานบริการสองรายการนี้:
    *   **Build > Authentication**: คลิกเริ่มต้นใช้งาน (Get Started) และเปิดใช้งาน **Google** ในหน้าหลักวิธีล็อกอิน (Sign-in providers)
    *   **Build > Firestore Database**: กดปุ่มสร้างฐานข้อมูล (Create Database) แนะนำให้ใช้ตัวเลือกเซิร์ฟเวอร์พื้นที่ครอบคลุมภูมิภาคเอเชีย เช่น `asia-southeast1` (สิงคโปร์)
4.  **เพิ่มสิทธิ์ความปลอดภัยให้โดเมน (Authorized Domains) - สำคัญมาก!**:
    *   ในเมนู **Authentication** -> เลือกแถบ **Settings** (การตั้งค่า) ด้านบน
    *   คลิกที่เมนูย่อย **Authorized domains** (โดเมนที่ได้รับอนุญาต) ทางขวา
    *   กดปุ่ม **Add domain** (เพิ่มโดเมน) และพิมพ์โดเมนเหล่านี้เพิ่มเข้าไปทีละหนึ่งรายการ:
        1. `localhost` (สำหรับจำลองรันบนคอมพิวเตอร์ส่วนตัว)
        2. โดเมนของ Vercel ของคุณ (เช่น `your-app.vercel.app` และโดเมนอื่นๆ ที่ Vercel สุ่มมอบให้ช่วง Deploy)
        3. `ais-dev-j3z544l2rahfuh6vzuix2s-542200585091.asia-east1.run.app` (หากคุณต้องการทดลองเทสระบบเชื่อมหน้าล็อกอินผ่านหน้าต่างพรีวิวฝั่ง AI Studio ด้วย)
5.  **อัปโหลดกฎการเข้าถึงข้อมูล (Firestore Rules)**:
    *   ไปที่เมนู **Firestore Database** -> แถบ **Rules** (กฎ) ด้านบน
    *   คัดลอกรหัสกฎความปลอดภัยทั้งหมดจากไฟล์ `firestore.rules` ในโฟลเดอร์โปรเจกต์นี้ ไปวางทับหน้าจอ จากนั้นกดปุ่ม **Publish** (เผยแพร่) ตัวกฎจะบังคับใช้คุ้มครองข้อมูลทันที

## ⚙️ ขั้นตอนที่ 2: ทำการตั้งค่าการเชื่อมต่อในส่วนเว็บแอปของคุณ

เพื่อจูนแอปพลิเคชันของคุณให้คุยกับเซิร์ฟเวอร์ Firebase ตัวส่วนพอร์ทัลของคุณโดยตรง:
1.  ในหน้าแรกของ **Firebase Console** -> กดไอคอนฟันเฟืองด้านข้างเมนู Project Overview เลือก **Project Settings**
2.  เลื่อนลงมาด้านล่างสุดในหัวข้อ **Your apps** -> เลือกไอคอน `</>` (เว็บแอป) เขียนชื่อแอปของคุณ และเลือก "Register app"
3.  คุณจะได้หน้าต่างแสดงชุดอ็อบเจกต์ **Firebase SDK config** เช่น:
    ```json
    {
      "apiKey": "AIzaSy...",
      "authDomain": "your-app-id.firebaseapp.com",
      "projectId": "your-app-id",
      "storageBucket": "your-app-id.firebasestorage.app",
      "messagingSenderId": "...",
      "appId": "..."
    }
    ```
4.  **อัปเดตไฟล์คอนฟิก**: 
    *   เข้าไปแก้ไขไฟล์ `/firebase-applet-config.json` ในโปรเจกต์
    *   แก้ไขค่าคีย์ต่างๆ ภายใน JSON ให้ตรงกับค่าเว็บแอปใหม่ของคุณ (พยายามอย่าลืมอัปเดตค่า `"projectId"`, `"authDomain"`, `"apiKey"` และตรวจสอบให้แน่ใจว่าค่า `"firestoreDatabaseId"` สอดคล้องกัน)

## 🚀 ขั้นตอนที่ 3: กระบวนการลากไฟล์ขึ้นรันบน Vercel

เมื่อตั้งค่า Firebase เสร็จสิ้น การนำไปรันบน Vercel นั้นทำได้ในเวลาไม่ถึง 2 นาที:

### วิธีแบบเชื่อมคลัง Git ด่วนที่สุด (แนะนำ):
1.  นำรหัสซอร์สโค้ดของโปรเจกต์นี้ทั้งหมด เข้าไปเก็บไว้ในคลัง **GitHub / GitLab / Bitbucket** ส่วนตัวของคุณเอง
2.  เปิดหน้า **[Vercel Dashboard](https://vercel.com/)** แล้วเข้าสู่ระบบ
3.  กดโปรเจกต์ใหม่ **Add New > Project**
4.  เลือก Import คลังโค้ดจากบัญชี GitHub ที่พึ่งสร้างขึ้น
5.  ในแท็บตั้งค่ากระบวนการบิลด์ Vercel จะจดจำการตั้งค่า React-Vite โดยอัตโนมัติ:
    *   **Framework Preset**: มองหาเป็น `Vite` (มีสัญลักษณ์สายฟ้า)
    *   **Build Command**: `npm run build`
    *   **Output Directory**: `dist`
6.  คลิกปุ่ม **Deploy** แล้วรอประมาณ 30 วินาทีถึง 1 นาที ระบบจะสร้าง URL สดมาบริการให้คุณฟรีพร้อมใช้งานทันที!

*   **วิธีใช้งานได้อย่างราบรื่นสุดยอด**: ให้มองไปที่แถบพรีวิวรูปจำลองขวาเกือบบนสุด จะมีไอคอนไอคอนหนึ่งชื่อว่า **"Open in New Tab" (เปิดในแท็บใหม่)** 
    *   เมื่อคลิกเข้าไป แอปจะเปลี่ยนหน้ารันเป็นหน้าต่างแยกเดี่ยว (Standalone page)
    *   ระบบกลไกป๊อปอัพเข้าสิทธิ์ Google Sign-in จะทำงานได้อย่างสมบูรณ์แบบโดยไม่ต้องตั้งค่าใดๆ เพิ่มเติมในเบราว์เซอร์ของคุณ!
